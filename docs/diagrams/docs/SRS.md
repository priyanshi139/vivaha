# Software Requirements Specification (SRS)

## 1. Introduction

### 1.1 Purpose

This document describes the requirements of the "Vivaha - Wedding Planning App". It defines the system functionality, user interactions, and constraints for efficient wedding planning.

### 1.2 Scope

Vivaha is an application designed to simplify wedding planning by providing a centralized platform for managing vendors, budgets, guest lists, and tasks. It helps users reduce time, improve coordination, and organize all activities in one place.

### 1.3 Definitions

* Admin: Manages the entire system
* Vendor: Provides wedding services (catering, decoration, etc.)
* Couple: End users planning the wedding
* UI: User Interface

---

## 2. Overall Description

### 2.1 Product Perspective

Vivaha is an application developed using HTML, CSS, and JavaScript with dashboards for different user roles.

### 2.2 Product Functions

* User registration and login (Admin, Vendor, Couple)
* Vendor listing and management
* Booking system for wedding services
* Budget planning and tracking
* Guest list management
* Task checklist and moodboards
* Notifications and updates

### 2.3 User Characteristics

* Basic computer and internet knowledge required
* Suitable for non-technical users

### 2.4 Constraints

* Requires internet connection

### 2.5 Assumptions

* Users provide correct information
* Vendors update their services regularly

---

## 3. Functional Requirements

* The system shall allow users to register and login
* The system shall provide role-based access (Admin, Vendor, Couple)
* The system shall allow vendor management
* The system shall enable booking of services
* The system shall manage budget and guest list
* The system shall provide task checklist and planning tools
* The system shall send notifications to users

---

## 4. Non-Functional Requirements

* Performance: Fast response time
* Usability: Simple and user-friendly interface
* Reliability: System should work without failure
* Security: User data should be protected
